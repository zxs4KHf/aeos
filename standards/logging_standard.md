# AEOS 日志与可观测性标准 v2.0

Status: active

## 一、事件设计

- 日志记录可操作事件，而不是无差别输出内部状态。
- 使用项目现有级别和结构；适用时包含时间、组件、事件名、关联 ID 和错误上下文。
- DEBUG 用于诊断细节，INFO 用于重要正常状态变化，WARN 用于可恢复异常，ERROR 用于需要调查的失败。

## 二、安全与隐私

- 不记录凭证、会话令牌、完整个人数据或不必要的请求正文。
- 对外部输入和错误对象进行安全序列化，避免日志注入和意外递归。
- 根据数据分类设置访问、保留和删除策略。

## 三、容量与可靠性

- 文件轮转、集中采集、采样和保留阈值由流量、磁盘预算和合规要求决定，不使用通用固定大小。
- 日志系统失败不应静默阻塞核心业务；安全审计日志除外，其失败策略应被明确设计。

## 四、验证

对关键告警和故障路径验证日志是否包含足够诊断信息，同时确认敏感字段已脱敏。指标和 trace 在比日志更适合表达连续值或调用链时优先使用。
