# 📋 AEOS 日志记录规范 (Logging Standard)

本规范规定了日志格式、日志级别分配以及日志文件的容量轮转和自动清理标准。

---

## 一、 日志分级与使用规范

所有的运行时日志输出必须使用固定的分级，并标明时间戳与产生组件模块的标签（Tags）：

1. **`[DEBUG]`**：仅在本地调试时开启的精细化排查日志，包含具体的底层 SQL 查询语句、API 响应的冗余数据等。生产环境下默认不输出。
2. **`[INFO]`**：一般正常业务事件记录（例如：`[ReminderManager] Syncing 31 reminders...`）。
3. **`[WARN]`**：业务存在异常但系统可容错处理（例如：检测到本地锁文件过期、数据库在重试后恢复连接等）。
4. **`[ERROR]`**：影响系统核心逻辑运行的严重失效（例如：飞书 WebSocket 秘钥错误、API 请求超时熔断等）。必须包含错误堆栈（Stack Trace）。

---

## 二、 日志文件容量轮转与清理 (Rotation)

1. **大小阈值控制**：
   - 本地日志文件（如 `logs/bot_running.log`）必须在写入时进行大小校验。当单个日志文件体积**超过 1MB** 时，系统必须自动执行日志容量截断（Truncation）或进行容量轮转重命名备份（如 `.log.1`）。
   - 严禁使本地日志无限制写入占用磁盘空间。
2. **敏感信息脱敏**：
   - 写入日志文件的数据必须过滤掉明文 Token、AppSecret 及用户的私密信息。
