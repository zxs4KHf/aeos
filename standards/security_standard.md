# 🔒 AEOS 安全规范 (Security Standard)

本规范规定了敏感信息管理、输入验证以及越权操作的防护底线。

---

## 一、 敏感凭证与环境变量管理

1. **凭证隔离与防泄漏**：
   - 严禁在任何代码文件、注释、甚至是 Markdown 设计文档中直接硬编码敏感凭证（如 AppSecret, API Key, 数据库密码）。
   - 所有的凭证必须存储在本地未提交的 `.env` 环境变量配置文件中，通过 `process.env` 进行安全读取。
2. **Git 仓库黑名单拦截**：
   - 必须在 `.gitignore` 中显式添加 `.env` 及其各类变种。
   - 智能体在提交代码前，需确认修改过的代码文件行内不包含明显的 Token、Key 或明文密码。

---

## 二、 输入校验与命令执行防范

1. **防范命令注入 (Command Injection)**：
   - 智能体自动执行 shell 命令时，必须保证参数是安全转义过的。
   - 尽量使用 Node.js `child_process.execFile` 或 `spawn` 传递参数数组，而不是在 `child_process.exec` 中直接拼接外部传入的字符串。
2. **数据库输入过滤**：
   - 对 SQLite 或外部数据库进行查询时，必须使用参数化查询或预编译语句，严禁直接通过 SQL 字符串拼接传递输入数据，以防止 SQL 注入。
