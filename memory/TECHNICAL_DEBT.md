## 二、 债务自查与 DoD 准入机制 (DoD Gate)

任何新功能开发在标记为 Done 并发布前，智能体**必须**对照以下机制自查：
1. **是否存在遗留 TODO**：检查提交的代码中是否存在未完成的 `// TODO` 或 `// FIXME`。如果有，必须将其整理录入本文件表格中，或当场解决。
2. **是否引入了新的依赖债务**：是否有未添加至 `package.json` 的隐式外部包，或者是否有容易产生版本冲突的库升级？
3. **测试覆盖率滑坡**：新增逻辑是否低于 `> 80%` 的单元测试线？如果低于，必须补充用例。
